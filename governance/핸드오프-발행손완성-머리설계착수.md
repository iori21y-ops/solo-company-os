## 2. 마스터플랜 — 확정된 방향
**목표:** 기존 n8n GWP는 **참조용 설계도**로만 쓰고(절대 수정 안 함), 새로 만든 에이전트 플로우(파트C) 위에 워드프레스 자동발행을 **에이전트 cycle로 새로 짓는다.** 최종적으로 모든 업무를 이 에이전트 기반으로 운영. 만든 뒤 **기존 n8n과 맞붙여 "어느 쪽이 더 나은지·어떻게 고칠지" 비교 분석**하는 게 종착점.

**확정된 역할 분담(경년님이 직접 바로잡은 핵심):**
- **발행 = marketing 에이전트 일** (content 아님). content는 검수통과본까지만.
- **전체 cycle 체인 발동·조율·결재 = 관리팀장(opus).** 개별 실무 에이전트가 전체를 컨트롤하지 않음.
- 경계선: GWP-2 해부 결과 `IF 통과?` 노드 직후 = content→marketing 경계 = **결재 게이트 자리**.

```
[관리팀장 = 전체 cycle 체인 발동·조율·결재판]
   ├─ data       → raw 기사 수집·정합성 → approved
   ├─ content    → 주제선정 → 초안(Claude, B안) → 검수 → 검수통과본 → approved
   │   ★ 결재 게이트 (관리팀장 품질검토 → 정기=관리팀장 / 고위험=대표)
   └─ marketing  → 발행(CTA·이미지·WP) → 성과측정 → cycle 종료
```

## 3. 이번 세션 확정 사실 (사실로 검증된 것만)

### 3-1. GWP-1/2 가동 메커니즘 규명 (의문 해소)
- **GWP-1**(active=False): 새벽3시 트리거 → curated_topics 채움 → HTTP로 **GWP-2 activate** → executeWorkflow로 GWP-2(2b157476) 호출.
- **GWP-2**(active=True): 5분마다 트리거, 큐 소진 후 **자가 deactivate**(HTTP). 반려 시 GWP-3(b2517adf) 호출.
- **정정:** 직전 핸드오프는 "파이프라인이 돈다"고 했고, 내가 처음엔 "멈춰 있다"고 단정했으나 — **경년님 정정: 실제로 발행되고 있음.** 워크플로별 트리거·스케줄이 제각각이라 부분적으로 돌아감. (GWP-1만 보고 전체를 추론한 내 단정이 틀렸음.)

### 3-2. GWP-2 전체 토폴로지 (76노드) — content/marketing 경계 확정
```
[CONTENT] 입력어댑터→브리프→Writer프롬프트→퓨샷→Obsidian검색→Claude초안
          →초안추출→검수프롬프트→Gemini검수+SEO→검수파싱→피드백저장→[IF 통과?]
   ═══ ★결재 게이트 들어갈 자리 (현재 게이트 0 = 검수통과 즉시 자동발행) ═══
[MARKETING] CTA+UTM→이미지(Pexels/DALL-E)→WP미디어업로드→HTML이미지삽입
            →WordPress발행→RankMath SEO→published→Telegram성공
   반려 시: 수동검토큐→GWP-3(WF-2R) 재작성 루프
```
- 군더더기로 버릴 것: **자가 activate/deactivate 자기제어 하나뿐.** 나머지는 다 일하는 근육.
- **초안 엔진 결정 = B안 확정**: 초안은 Claude(유료, 품질 우선) 유지. 발행량은 결재 게이트로 통제. (과거 4월 Ollama 폴백 시 품질 1.8/5로 추락한 이력 때문에 로컬로 안 내림.)

### 3-3. 발행 "손" 완성 — wp_publish.py (실발행 검증 완료) ⭐핵심 산출물
- **경로:** `~/projects/_meta/agents/marketing/scripts/wp_publish.py` (2,567 bytes)
- **소유:** marketing 에이전트 (경로로 경계 박음)
- **방식 (C안 = 새 발행 스크립트):** n8n 발행 구간을 빌리지 않고 파이썬으로 직접 WP REST API 호출. 이유 = n8n 발행을 빌리면 비교 분석이 안 됨(발행부가 동일해짐).
- **인증:** `~/projects/cadam/cadam-n8n/.env`에서 재사용 — `WP_URL=https://cadam21y.mycafe24.com`, `WP_USER=cadam21y`, `WP_APP_PASSWORD`(앱비번, administrator 권한 인증 성공 검증됨).
- **엔드포인트:** `/wp-json/wp/v2/posts` (Basic Auth, 앱비번 공백 제거 처리).
- **안전장치:** 기본 `draft` 발행. `--publish` 플래그 있어야만 즉시 공개.
- **사용법:** `python3 wp_publish.py --title "제목" --file 본문파일 [--publish]`
- **검증:** 테스트 글 **ID 449** draft 발행 성공 (https://cadam21y.mycafe24.com/wp-admin/post.php?post=449&action=edit). 사이트엔 안 보이는 상태. 삭제 보류 중(비교용 + 영구삭제는 경년님이 직접).
- **MVP라 의도적으로 뺀 것:** 검수, 이미지, CTA/UTM, RankMath SEO, 재작성 루프. → 나중에 붙일 때 "n8n 통짜 vs 에이전트 단계추가" 비교 포인트.
