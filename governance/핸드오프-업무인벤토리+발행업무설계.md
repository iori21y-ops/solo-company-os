## 2. 마스터플랜 — 목표가 바뀐 지점 (중요)
- 처음엔 "현재 업무를 라벨링(인벤토리)"이 목표였다.
- 조사 중 드러난 사실: 현재 발행 파이프라인(n8n GWP 체인)은 **에이전트 도입 전에 만든 모놀리식**이다. 모든 단계(크롤→큐레이션→글쓰기→검수→발행→최적화)가 n8n 워크플로 안에 통째로 박혀 있고, 중간 결재가 없다.
- **대표 판단(경년님):** 이 n8n 워크플로는 "돌긴 하지만 마음에 안 든다 — 쓸데없는 게 많고 품질이 떨어진다." 따라서 **보존 대상이 아니라 참조용 설계도**다.
- **다음 세션 미션:** n8n GWP를 *참조*해, 필요한 단계만 취하고 군더더기는 버려, 파트C 에이전트 롤(data/content/marketing + 결재 게이트 + 학습루프)에 맞는 **새 워드프레스 자동발행 주기 업무**를 설계한다.

### 4-4. GWP 체인 end-to-end 흐름 (참조 설계도)
```
RSS크롤(data) → 기사풀(supabase)
  → GWP-1 인사이트추출·큐레이션 → curated_topics       [content]
  → GWP-2 브리프→초안→검수+SEO                          [content]
     · 검수통과 ─▶ CTA·UTM→이미지→WordPress발행→RankMath [marketing]  ← 경계선
     · 검수반려 ─▶ GWP-3 재작성 루프(1차Gemini→2차Sonnet) [content]
  → published → GWP-5 성과측정 → 피드백                  [marketing→측정]
  → GWP-4(Writer)·GWP-6(검수) 프롬프트 자기개선          [content 학습루프]
WF-7 인사이트 머지(매일6시, 병렬)                          [content/data 가공]
```
- **content↔marketing 경계 = GWP-2 내부 "검수통과 → 발행" 사이.** 여기 현재 결재 게이트 0(5분마다 자동발행). 파트C는 여기에 "발행 직전 게이트2(채널 강도: 자사 약/네이버·인스타 강)" 필요.
- **좋은 발견:** 학습루프가 이미 구현돼 있음(GWP-4/5/6, WF-7). 파트C §4 4계층과 대응 → 새로 안 만들고 매핑·개선만.

### 4-5. 미해결 의문 (다음 세션 첫 확인)
- **GWP-1(큐레이션 엔진)이 비활성**인데 GWP-2는 그 산출물(curated_topics)을 읽음. 설계상 GWP-1(새벽3시)→GWP-2 활성화→GWP-2 발행 후 자가비활성. 즉 **GWP-1 꺼져 있으면 일일 자동 사이클이 안 돎** → 현재 전체 GWP가 수동 기동 상태일 가능성. 실제 가동 여부 확인 필요.
