# 관리팀장 도구 개방 검토 (이어가는 중)

## 1. 이어가는 중
새 작업 아님. 2026-05-30 디스코드 2-봇 인프라+상호호출 완성 후, 다음 단계로 "관리팀장(Claude Code)에게 도구를 열어 실제 작업을 시킬지" 검토부터 시작한다. 이번 세션은 설계/안전 검토 먼저, 구현은 합의 후.

## 2. 큰 그림 / 왜
OpenClaw 강의 구조로 디스코드 2-봇 협업 구축 완료:
- 기획팀장(OpenClaw/GPT-5.5) = 총괄/구현
- 관리팀장(claude-broker/Claude Code) = 리뷰어
당초 Claude를 총괄로 쓰고 싶으나 과금0 때문에 불가로 결론냈으나, 새 사실 발견: 관리팀장은 맥미니 Claude Code를 구독(과금0)으로 부르고 있고, 지금 --tools "" 로 도구만 꺼둔 상태다. 이 옵션을 풀면 과금0 유지하면서 Claude가 파일읽기/코드수정/명령실행까지 가능. 즉 리뷰어를 넘어 구현자로 승격 가능. 이게 이번 검토 주제.

## 3. 이미 구축된 자산 (전부 가동 중)
서버 ID 1510170583265050744, #작업실1 채널 1510170922005565470.
- 기획팀장 = OpenClaw봇(봇ID 1510185115479965738), gpt-5.5, 설정 ~/.openclaw/openclaw.json. requireMention:false, allowBots:mentions, ignoreOtherMentions:true, guilds.users에 으라차+관리팀장봇ID. LaunchAgent ai.openclaw.gateway.
- 관리팀장 = claude-broker(관리팀장#6993). 위치 ~/.openclaw/workspace/claude-broker. Node.js 직접작성. claude -p CLI subprocess 호출. claude 절대경로 /Users/kim/.local/bin/claude. CLAUDE_FORCE_OAUTH=1 + CLAUDE_CODE_OAUTH_TOKEN = 구독 호출 과금0(맥미니 env에 ANTHROPIC_API_KEY 없음). LaunchAgent com.rentailor.claude-broker.plist KeepAlive.
  - 핵심: runClaudeCli가 --tools "" 로 호출 → 도구 전부 꺼짐 → 파일/코드/명령 불가, 순수 대화만. 이게 도구 개방의 해제 대상.
  - .env: ALLOWED_GUILD_IDS/CHANNEL_IDS/USER_IDS(으라차)/ALLOWED_BOT_IDS(기획팀장봇), SYSTEM_PROMPT(리뷰어+승인/반려 지시), BOT_PREFIX 제거됨(멘션전용).

## 4. 멘션 규칙 (검증 완료)
- 으라차 일반채팅 → 기획팀장만 반응
- @관리팀장(반드시 멤버그룹 봇계정, 역할 아님) → 관리팀장만 반응, 으라차에게 답
- 기획팀장이 @관리팀장 봇멘션 → 관리팀장 반응, @기획팀장 붙여 응답 → 기획팀장 수신 (봇 핑퐁 작동 확인)
- 주의: 디스코드에 봇과 동명 역할 존재. 멘션 시 멤버그룹 봇계정 선택 필수(역할 멘션은 broker가 못 받음).

## 5. 이번 세션 주제 = 도구 개방 검토 (구현 아님)
관리팀장 broker의 --tools "" 를 풀어 Claude Code 실제 능력을 줄지 검토.
리스크: ①broker는 도구차단 전제 설계, 도구 열면 디스코드 메시지(외부 신뢰불가 입력)로 맥미니 파일/명령 실행 가능 → prompt injection 위험 ②작업 디렉토리/명령 범위 미정 → 맥미니 전체 노출 ③봇 핑퐁+도구+5회종료 미구현 = 폭주 위험.
검토: ①--tools로 특정 도구만 선별 허용 가능한가 ②cwd 샌드박스 ③허용 명령 화이트리스트 ④승인 게이트 ⑤--permission-mode 등 권한 플래그.

## 6. 우선순위
이번 세션은 도구 개방 안전설계 검토+합의까지. 구현은 합의 후. 다음: 5회카운터+승인/반려 종료, 봇토큰 재발급.

## 7. 첫 지시 (조사/검토만, 코드수정 금지)
1. claude -p 도구/권한 옵션 전부 조사: --tools, --permission-mode, --allowedTools, --add-dir 등. claude -p --help 및 공식문서. 추측금지.
2. broker 도구 열었을 때 맥미니 노출 범위 분석(cwd 어디인지).
3. 안전한 도구 부여 설계안 1개 제시(디렉토리 제한+도구 선별+injection 방어).
4. 도구 개방 시 핑퐁 폭주 위험과 5회종료 선후관계 검토.
5. 코드/설정/git 수정 금지, 조사보고서만. 복잡하면 어드바이저(Opus) 의견요청.
