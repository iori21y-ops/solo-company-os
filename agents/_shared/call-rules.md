| 에이전트 | 플랫폼 | 역할 ID | 호출 방법 |
|---------|--------|---------|----------|
| 관리팀장 | broker | 1510185782822830211 | 역할멘션 / [CALL:관리팀장] |
| 마케팅 | broker | 1510266153208119398 | 역할멘션 / [CALL:마케팅] |
| 콘텐츠 | broker | 1510282095539519673 | 역할멘션 / [CALL:콘텐츠] |

## 호출 방법
### A. broker 봇이 다른 에이전트를 호출할 때
- 응답 본문과 별도 줄에 `[CALL:에이전트명] 요청 내용` 작성
- broker는 `[CALL:]`을 해당 에이전트의 디스코드 역할멘션으로 변환해 발송한다
- 자기 자신은 호출하지 않는다
- 한 응답에 여러 에이전트 호출 가능 (각각 별도 줄)
- 직접 처리 가능한 작업은 호출하지 말 것
- 호출 없는 일반 응답에는 [CALL:] 태그를 쓰지 않는다
- 다른 에이전트가 부른 업무에 답하고 더 넘길 호출·결재·대표 결정 요청이 없으면 마지막 줄에 `종료`를 쓴다

### B. 직접 역할멘션
- 디스코드 역할 멘션 `<@&역할ID>`도 유효하다
- broker 응답 본문 안의 위 표 역할멘션은 실제 ping이 허용된다
- 예시: `<@&1510179969354170481> 이번 달 콘텐츠 기획안 검토 요청`

### C. OpenClaw 봇이 다른 봇 호출할 때
- 디스코드 역할 멘션 `<@&역할ID>` 사용
- sessions 사용 금지 (세션 ID 불일치로 실패함)

## 종료 규칙
- 봇 간 상호 호출에는 답변 횟수 제한을 두지 않는다
- 대신 각 에이전트는 자기 차례의 처리가 끝났고 추가 호출이 없으면 마지막 줄을 `종료`로 닫는다
- `종료`는 해당 응답 체인이 끝났다는 신호다. 새 지시는 대표님 또는 새 역할멘션/호출로 다시 시작한다

## 업무 칸반 기록 규칙 ([TASK])
실질 업무를 시작/완료할 때 ERP 업무 칸반에 카드를 남긴다. broker가 이 태그를
감지해 대시보드(/api/tasks)에 기록한다. (담당자 이름은 broker가 자동 기입)

- 착수 시: 응답 본문과 별도 줄에 `[TASK:진행중] 업무제목 | 출처 | 우선순위`
- 완료 시: 별도 줄에 `[TASK:완료] 업무제목`
  ⚠️ 완료의 제목은 착수 때와 **글자까지 동일**해야 한다. 다르면 카드를 못 찾는다.
- 보류/취소: `[TASK:보류] 제목` / `[TASK:취소] 제목` (주로 관리팀장)
- 산출물 있으면 뒤에 ` | result=경로` (예: `[TASK:완료] 엔카 시세수집 | result=agents/data/outputs/approved/verdict-x.md`)

값: 출처=`스팟`|`주기`|`디스코드`, 우선순위=`높음`|`보통`|`낮음`(생략 시 보통)
원칙: 산출이 있는 업무 단위에만. 연 카드는 반드시 완료/보류/취소로 닫는다. 순수 보고엔 쓰지 않는다.
