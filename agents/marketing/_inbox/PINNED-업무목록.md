# 📌 마케팅 업무목록 — 스레드 고정용

## 🔄 주기 업무 3종 (regular)
| 코드 | 업무 | 게이트 | 발행결재 |
|------|------|--------|----------|
| 주기A | 확정 발행계획 실행 — content approved + data 합본 → 발행물 제작 → 발행 → _shared/final | 게이트2(발행 직전·필수) | 자사블로그=관리팀장 / 네이버·인스타·쇼츠=대표 |
| 주기B | 신규 주제 발굴 — 스카우트+sales전환+트렌드 → 5질문 Tier 판정 → 발행계획 확정 | 게이트1(Tier 사람승인) | — |
| 주기C | D+7 노출측정·환류 — 발행 7일 후 지표측정 → §5기록 → sales 핸드오프 → 주기B Q4 환류 | 없음(보고형) | — |

## ⚡ 스팟 업무 4종 (adhoc) — _inbox/adhoc-queue.md 적재
- 스팟1 트렌드 즉응 / 스팟2 대표 직접지시 / 스팟3 시즌 캠페인 / 스팟4 광고·랜딩 카피
- 외부 노출은 긴급해도 발행 직전 게이트 필수 (결재 생략 금지)

## ▶️ 자동화 현재 상태 (정직 표기)
- **정의:** 완료 (SKILL.md 확정)
- **첫 사이클:** 관리팀장이 수동 디스패치 (주기A·B 착수 지시 완료)
- **시간 스케줄러(크론):** ❌ 미구축 — "발행 배선" 과제(등록만 됨, 구현 미착수)
- **현재 자동 의미:** "트리거되면 게이트 직전까지 자율 실행 → 결재카드 상신"까지. 스스로 시간 맞춰 굴러가지는 않음.

## 🚦 불변 원칙
**자동 ≠ 무결재 발행.** 외부 노출은 회수 불가 → 발행 직전 게이트 항상 유지.
