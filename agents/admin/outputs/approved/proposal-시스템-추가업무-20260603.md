# 시스템(ops) 추가 업무 분석

## 전제 (사장님 제시)
- 시스템 = 로컬AI → **무비용**, 그래서 일을 많이 시키고 싶다.
- 단 **역할 한계**(관찰·체크·보고만, 복구·집행 X)와 **LLM 성능 한계**(약한 추론)가 있다.

## 설계 원칙 (한계를 비용으로 바꾸지 않기)
1. **스크립트가 계산, LLM은 서술만**. 판정·임계비교는 결정론 스크립트가 한다. 로컬LLM은 "요약문 작성·예외 분류"만. 약한 모델에 판단을 많이 시키면 오탐이 쏟아진다.
2. **무비용의 이점은 "스크립트 확장"으로 회수**한다. 신뢰성 있는 결정론 cron을 많이 돌리는 것이지, LLM 판단 호출을 늘리는 게 아니다.
3. **예외만 보고 + 1일/1주 다이제스트**. 매 이벤트 알림 금지(디스코드 홍수 방지). 기존 SYS의 "이상 시만/실패 시만" 규칙 계승.
4. **관찰·보고 경계 불변**. 변경·삭제·재시작·재시도 판단은 전부 관리팀장(ADMIN-02, 대표 승인 후).

## A. 지금 추가 가능 — 관찰 전용 (경계 안전, LLM 얇음)
| 후보 | 내용 | LLM 몫 |
|------|------|--------|
| SYS-A1 헬스체크 상시화 | SYS-01을 일1회→시간별 watch, **임계 초과만** 보고 | 없음(스크립트) / 일1회 요약만 |
| SYS-A2 엔드포인트 핑 | broker·OpenClaw·md-viewer(9724)·ollama(11434)·n8n 가용성 주기 핑 | 없음 |
| SYS-A3 자원 추세 적재 | 디스크·메모리 시계열 누적 → 임계 도달 예측 보고 | 추세 한 줄 서술 |
| SYS-A4 만료 카운트다운 | Tailscale 키·API 토큰·구독 갱신일 D-day 추적(SYS-04' 보강) | 없음 |
| SYS-A5 파일명 규칙 검사 | `{유형}-{주제}-{YYYYMMDD}.md` 위반 **목록만** 보고(관리팀장 Rule Enforcer 보조) | 없음(순수 스크립트) |
| SYS-A6 pending 나이 스캔 | 48h 초과 파일 리스트 → ADMIN-04 입력 공급 | 없음 |
| SYS-A7 깨진 링크·경로 검사 | vault 내 상대링크·approved 참조 깨짐 목록 | 없음 |
| SYS-A8 폴더 교차침범 검사 | git diff로 "남의 outputs/ 수정" 탐지 → 보고(경계규칙 보조) | 없음 |
| SYS-A9 외부소스 변동핑 | 환율·보조금·정책 페이지 해시 비교, 변동 시 **data로 핸드오프**(DATA-04 트리거 공급) | 없음 |
| SYS-A10 운영메트릭 적재 | uptime·실행성공률·토큰량·pending수 1줄 시계열 → ADMIN-03 익일보고 raw 공급 | 다이제스트 서술 |

> A5·A6·A7·A8은 **LLM 0% 순수 스크립트**다. 약한 모델 리스크 없음 → 가장 먼저 켜도 안전.

## B. 가능하나 승인 게이트 필수 — 상태 변경 동반 (경계 주의)
> ⚠️ 아래는 **변이(mutation)** 라 이번에 새로 정한 "복구·집행=관리팀장(대표 승인 후)" 경계와 충돌할 수 있음. 시스템은 **후보 목록만 보고**, 실행은 관리팀장 경유.
| 후보 | 변이 성격 | 안전화 |
|------|-----------|--------|
| 크롤러 재시도 | 자율 재시도=복구 판단 | "실패 보고만" 또는 **사전승인된 결정론 N회 재시도**(LLM 판단 X) |
| 로그 로테이션·임시파일 정리 | 삭제=변이 | "정리 후보 목록만 보고"로 격하(A 방식) |
| git 스냅샷/auto-commit | vault 상태 변이 | 관리팀장 승인 후 집행, 또는 보고만 |

## C. 제외 — 로컬LLM에 부적합 (Opus 에이전트 유지)
- 복구 판단·원인 진단 → 관리팀장(ADMIN-02)
- 품질·팩트 검토 → 관리팀장/data
- 콘텐츠 생성·큐레이션·우선순위 → content/marketing/strategy

## 중복 회피 메모
- 비용 감시(구 SYS-04)는 **이번 세션 폐기** → 비용형 신규 제안 없음.
- 토큰 추세(SYS-04')는 **존재** → A10은 신규 중복이 아니라 SYS-04'/ADMIN-03에 **데이터 공급**으로 연결.
