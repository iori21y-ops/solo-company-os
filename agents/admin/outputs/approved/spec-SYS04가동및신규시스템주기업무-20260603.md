# SYS-04 가동안 + 시스템 신규 주기업무 제안

> **원칙 준수**: 시스템(ops)=**관찰·체크·보고만**. 복구·집행은 ADMIN-02(대표 승인 후). 새 메커니즘 금지 → 기존 보고 루프·헬스체크 스크립트·결재 게이트 **재사용**.

---

## 1. SYS-04(비용 알림) 가동안 — "정의됨·미가동 → 운영중"

### 현실 진단 (블로킹 사유)
- 맥미니 로컬에 **비용/토큰 usage 측정 소스가 없음**(broker·agent 토큰 로그 부재, billing은 전부 외부 대시보드).
- 따라서 **완전 자동 임계감지 알림은 현시점 구현 불가**. 데이터 소스 결정이 선행되어야 "라이브"가 됨.

### 가동 방식 — 2단계
| 단계 | 방식 | 구현 | 데이터 소스 |
|------|------|------|------------|
| **A안 (즉시 가동, 권장)** | **월간 수동집계 비용 리포트** | ops가 매월 2~3일 Anthropic·OpenAI·Gemini·Supabase·Vercel 대시보드 실측 → 임계 대비 보고 | 수동(대시보드 열람) |
| **B안 (후속)** | 자동 usage-API 폴링 | 각 벤더 billing/usage API 키 발급 → 일일 폴링·임계감지 알림 | usage-API |

- A안은 **오늘 바로 가동 가능**. SYS-01 헬스체크의 "이상 시만 보고" 패턴 계승(임계 초과 시만 알림).
- 측정 항목: 월간 ① Anthropic(broker+agents) ② OpenAI(OpenClaw) ③ Gemini(추출) ④ Supabase ⑤ Vercel — 합계 및 항목별.

### 제안 임계값 (기본값 — 승인 시 즉시 발효)
| 항목 | 🟡 주의(보고) | 🔴 경고(즉시 알림) |
|------|--------------|-------------------|
| 월 합계 | 예산의 80% | 예산의 100% 초과 |
| 단일 벤더 | 전월 대비 +50% | 전월 대비 +100% |

> ⚠️ **월 예산 총액(USD)이 미정**입니다. 대표님이 금액만 지정하면 A안 즉시 발효. (제안 기본값: **월 $300** — 조정 가능)

## 2. 시스템 신규 주기업무 제안 — 3건 (기존 행과 비중복 검증 완료)

> 전부 **관찰·체크·보고**형. 복구 트리거는 ADMIN-02로 분리. SYS-01(디스크·메모리·포트), SYS-05(백업·로그정리)와 중복 없음 확인.

| 신규ID | 업무 | 주기/트리거 | 산출(보고) | 기존 비중복 근거 |
|--------|------|------------|-----------|-----------------|
| **SYS-06** | **스케줄 잡 실제 발화 감사**(launchd/cron이 예정대로 *실제로 돌았는지* exit-code·최근실행시각 점검) | 일 1회, schedule 07시(SYS-01에 합류) | 미발화·실패 잡만 보고 | SYS-05는 "야간 자동화 실패시 보고"=개별 잡 결과. SYS-06은 **스케줄러 자체가 잡을 띄웠는지** 메타 감사(누락 발화 포착) |
| **SYS-07** | **인증·만료 모니터**(Tailscale 키·TLS 인증서·도메인·벤더 API 키 만료 D-30 경보) | 일 1회(SYS-01 합류) | 만료 임박 항목만 보고 | SYS-01은 Tailscale *연결상태*만 확인. 만료일 추적은 미커버 |
| **SYS-08** | **Supabase Storage 용량·쿼터 모니터**(버킷 사용량·플랜 한도 대비) | 주 1회, schedule | 임계(80%) 초과 시만 보고 | SYS-01은 로컬 디스크(/)만. 클라우드 스토리지 쿼터는 미커버 |

- 모두 **SYS-01 헬스체크 스크립트에 체크 블록 추가** 방식(새 메커니즘 0). SYS-08만 별도 주간 스케줄.
- SYS-06~08도 동일하게 **복구명령 제안만, 자동실행 X** → 집행은 ADMIN-02.

## 3. 대표님 결정 필요 (3건)
1. **SYS-04 가동 방식**: A안(월간 수동집계, 즉시) 승인 여부 — 권장.
2. **월 비용 예산(USD)**: 임계 기준 금액. (제안 기본값 $300)
3. **신규 SYS-06/07/08 레지스트리 등재** 승인 여부.

## 4. 승인 후 집행 순서 (관리팀장)
1. 레지스트리 `_shared/recurring-tasks.md` 시스템 섹션: SYS-04 상태 `운영중`으로, SYS-06~08 행 추가.
2. `[CALL:시스템]` — SYS-04 A안 월간 리포트 1회 실행 + SYS-06/07 체크블록을 `server-health-check.sh`에 추가, SYS-08 주간 스케줄 등록.
3. 가동 검증(실제 산출물 1회 생성) 후 상태 확정. 결과는 익일 7시 대표 보고 포함.
