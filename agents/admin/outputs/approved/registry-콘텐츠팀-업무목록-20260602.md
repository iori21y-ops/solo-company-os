# 콘텐츠팀 업무 레지스트리 — 주기/스팟 분류 + 실행 스펙

> 지시: Phase B — n8n 혼재 워크플로의 LLM 부분을 에이전트로 이관. 콘텐츠팀 주기 업무를 데이터팀 레지스트리와 같은 포맷으로 단일화.
> 원칙(advisor 점검 반영, 데이터팀 레지스트리 계승): **새 메커니즘 금지.** 보고루프·결재 게이트·전제조건(green verdict)은 content/SKILL.md에 이미 존재 → 재사용. 이 문서가 채우는 공백은 ① 각 항목의 **실행 스펙(트리거/입력/산출/합격기준)**, ② 주기 항목의 **보고 cadence**.
> **GWP-1 이관 결정(2026-06-02, 대표 선택)**: "검증·게이트 소유권만 이관". 큐레이션 LLM 배치 스크립트(`cadam-pipeline/scripts/content/curator_agent.py`)는 **SYS가 그대로 실행**. content는 그 산출물(green 데이터)을 **재사용 에셋으로 가공 + 정확성 게이트를 소유**한다. n8n GWP-1 비활성화는 보류(대체 경로 실행·검증 후 1건씩).

## 보고 루프 (재확인 — content/SKILL.md 그대로, 신규 아님)
1. content가 항목 수행 → 산출물을 `content/outputs/pending/`에 저장(파일명 `{유형}-{주제}-{YYYYMMDD}.md`, 프론트매터 source_cycle·trigger_type 필수).
2. 순수 보고(후보 목록·진행·완료·자산 만료)는 **채널 직접 게시**(관리팀장 [CALL] 금지). 승인·결재 필요 시 `[APPROVAL:관리팀장] 제목 | 요약 | /절대경로`.
3. 관리팀장 검토 → green이면 경량 승인(approved/ 이동) / 이슈 시 [CALL:콘텐츠] 수정 또는 대표 결재.
4. 생산 자산은 `content/assets/cadam/index.md`에 등록(재사용 추적).

**전제조건(불변)**: 소스 제작은 해당 데이터 구역의 **최신 정합성 verdict가 green일 때만**. 미검증 데이터로 제작 금지. (업스트림 = 데이터팀 레지스트리 A-1·A-2·A-5 verdict)

- **발행 채널 맞춤 마감(SEO·CTA·플랫폼 포맷)** → marketing 담당. content는 재사용 가능한 에셋까지. (쌍1)
