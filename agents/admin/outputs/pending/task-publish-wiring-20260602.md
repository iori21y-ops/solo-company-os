# 별건 과제: 발행 배선 (Publishing Wiring)

## 배경
마케팅 직접 실행 루프(`제작 → 결재 → 발행 → 측정`)에서 **제작·결재 상신까지는 완전 자가 실행**이 되나,
**발행 단계가 에이전트와 미배선**이다. 대표 승인(2026-06-02)으로 본 과제를 백로그에 **등록**한다.

## 현황 (정직 표기)
| 채널 | 발행 수단 | 에이전트 배선 |
|------|-----------|--------------|
| 자사 블로그/WP | `marketing/scripts/wp_publish.py` (REST, 인증 확인됨) | ⛔ 미배선 |
| 네이버 블로그 | `cadam-naver` (post-blog.js, 쿠키 인증) | ⛔ 미배선 |
| n8n 파이프라인 | `cadam-n8n` (GWP/VWP 크롤+큐레이션+WP발행) | ⛔ 미배선 |
| 인스타 / 쇼츠 | 수단 없음 | 수동 |

## 과제 범위 (구현 시)
1. 결재 승인(approved/) 산출물 → 발행 스크립트로 전달하는 트리거 정의
2. 발행 직전 게이트2(필수)와 스크립트 실행 사이의 가드 로직
3. 발행 결과(URL·일시)를 cycle §4·주기C §5로 회수하는 경로

## ⚠️ 구현 착수 전 필수 확인 (전역 안전규칙)
- **본 문서는 "등록"까지만 승인됨. 배선 실행은 그 자체로 별도 승인 + 사전 확인이 필요하다.**
- n8n 워크플로우 활성화/변경, 서비스 재시작, WP 실발행은 전역 안전규칙상 사전 확인 필수.
- 외부 노출은 회수 불가 → 발행 직전 게이트는 자동화에서도 생략 불가.
